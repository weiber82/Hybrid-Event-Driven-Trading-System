Aiding Long-Term Investment Decisions with XGBoost Machine Learning Model
1. 研究背景與動機

金融市場的趨勢變化通常具有高度非線性，且受到多種特徵交互影響。傳統統計方法對於複雜訊號辨識能力有限，尤其在處理不平衡資料與標記噪聲時表現不佳。研究提出使用 XGBoost 進行市場趨勢分類，旨在改善投資決策的訊號品質。

2. 研究目的

本研究目標為建構一個以 XGBoost 為核心的市場趨勢辨識模型，協助判斷趨勢起點、終點與方向，並探討多階段模型方法是否能提升整體預測品質。

3. 研究方法與架構

研究採用兩階段模型架構：

ChangePoints 模型

用於偵測趨勢開始與結束位置

特徵包含：前後價格變化率、Volume 比率、High/Low 比值等

TrendOrFlat 模型

用於將趨勢區段分類為上升、下降或盤整

特徵包含：線性回歸斜率、R²、成交量斜率、趨勢長度等

為處理標記不一致問題，研究者分析不同人工標記者對趨勢定義的差異，進而提出資料清理流程。
為處理高度不平衡資料（趨勢端點遠少於其他資料點），採用特徵工程與特別取樣策略進行校正。

4. 數據來源與實作

使用長期歷史股票資料進行實驗

以多種傳統方法（包含迴歸模型）進行比較

透過交易模擬驗證模型在實際應用情境的表現

5. 研究結果

XGBoost 顯示出優於線性模型的趨勢分類能力

多階段架構改善了端點偵測與趨勢方向判斷

特徵重要度可有效指出驅動趨勢變化的主要因素

使用交易績效作為衡量指標時，模型在收益與風險表現均優於基準策略

6. 研究貢獻

提供 XGBoost 用於金融趨勢辨識的系統性架構

提出端點偵測與趨勢分類的兩階段方法

討論標記不一致與不平衡資料在金融資料集中的常見問題

以交易績效作為模型評估指標，提高實用性

7. 研究限制與未來方向

限制：

主要使用日資料，未涵蓋高頻場景

特徵工程依賴研究者手動設計

標記方法仍帶有主觀性

未來方向：

探索自動化特徵工程

延伸至高頻金融市場

引入不同資料來源以提升穩定性

8. 總結

本研究展示 XGBoost 在金融市場趨勢偵測上的適用性，並透過兩階段模型提升趨勢端點辨識與方向分類能力。研究同時強調在金融資料中，標記品質、不平衡資料處理與以績效為主的模型評估方式的重要性。

9. Applicability to This System（可用於本系統的參考重點）

此研究中的方法與結論可提供本系統 L0：事件偵測層 以下參考：

9.1 XGBoost 作為事件偵測模型的有效性

研究證實 XGBoost 適合處理具有非線性與高度噪聲的金融訊號，符合本系統在 L0 層以梯度提升模型辨識事件的需求。

9.2 多階段架構的可行性

研究採用「端點偵測 → 趨勢分類」的兩階段架構，其概念可延伸至：

事件觸發（Barrier Hit）

方向分類（Long / Short / Neutral）

與本系統的 L0 架構邏輯相近。

9.3 特徵工程指引（Microstructure + 技術指標）

研究所提出的特徵工程方式（如價格變化比率、High/Low 比值、成交量變化）可作為本系統高頻資料特徵的參考。

9.4 標記與資料不平衡處理

本研究強調標記方法與資料不平衡對模型造成影響，對於事件驅動模型的 Triple-Barrier 標記策略具有參考價值。

9.5 以交易績效作為模型評估指標

研究主張預測 accuracy 不足以評估金融模型，應以交易績效衡量，本系統的回測架構亦可採用此標準。
