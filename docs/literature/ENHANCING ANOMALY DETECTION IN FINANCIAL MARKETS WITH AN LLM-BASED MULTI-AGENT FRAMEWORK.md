文獻導讀 https://arxiv.org/html/2403.19735v1

Park, T. (2024). Enhancing Anomaly Detection in Financial Markets with an LLM-Based Multi-Agent Framework. arXiv:2403.19735

1. 研究背景與動機

金融市場的異常偵測（anomaly detection）長期仰賴統計方法與深度學習模型。然而，即使能偵測出異常點，仍需要大量人工介入去檢查資料是否錯誤、事件是否合理、異常是否具意義。
該研究指出：

異常偵測的難題不僅在於偵測本身

更在於「異常後續的驗證與詮釋流程」仍高度依賴人工
因此作者提出使用 大型語言模型（LLMs）與多代理人（multi-agent）協作框架，模擬人類分析師的驗證流程。

2. 研究目的

本研究旨在建立一套多階段、多代理人的 AI 系統，
用於 協助驗證、解釋與分類金融市場資料中的異常事件，
並模擬現實中分析師與管理階層的討論機制。

3. 研究方法與系統架構

作者提出一套 五階段 multi-agent workflow，由 LLM 驅動：

3.1 異常偵測階段

使用統計方法（例如 Z-score）從日資料中找出極端異常值

選取 S&P 500 指數（1980–2023）作為示例

3.2 Data Conversion Agent

將表格資料與 metadata 轉換成 LLM 能理解的結構化問題

自動生成針對每個異常點的查詢問題

3.3 專家代理人（Expert Agents）

包含三種類型：

Web Research Agent：搜尋新聞/公告查證事件背景

Institutional Knowledge Agent：使用預先匯入的知識庫提供背景脈絡

Cross-Check Agent：比對其他資料來源（如 Yahoo Finance）驗證數值正確性

3.4 Summary Agent

整合多個專家代理人的回答

自動生成單一摘要報告

3.5 Management Discussion Agents

多個高階代理人（LLM）閱讀摘要

自動模擬管理層的討論、質疑與共識形成

最終產生決策或結論

以上構成一套近似人類分析流程的 AI pipeline。

4. 數據來源與實作

使用 Bloomberg 的 S&P 500 日資料

偵測三個歷史上已知的重大異常：

1987 Black Monday

2008 金融危機反彈

2020 COVID 崩盤

實際展示各代理人對事件的處理與回覆

展示管理者討論（LLM 之間的對話）

5. 研究結果

多代理人系統能有效檢查異常事件是否正確、提供背景原因

web research、知識庫與 cross-check 三種方法互補

管理層代理人能產生一致結論

整體能模擬人類分析師的「驗證」工作流程

提升資料品質審核效率、降低人工需求

6. 研究貢獻

作者的主要貢獻包含：

提出 金融異常事件分析的 LLM 多代理人框架

展示如何將 tabular data + metadata 轉換為語言模型可處理的結構

以代理人分工方式模擬人類分析師流程

引入管理層代理人，用於「共識形成」

實際驗證此流程能有效處理金融資料異常解讀

7. 研究限制與未來方向

論文提出以下限制：

僅使用日資料，未驗證高頻資料的能力

專家代理人的知識庫品質影響結果

web search agent 受限於搜尋結果與模型能力

目前仍需人類在系統設計上做監督
未來方向包含：

引入更高頻的金融資料

改進 metadata 管理

增強代理人的自動化能力

減少人工介入的部分

8. 總結

此研究展示出一個完整可重複的 LLM 多代理人流程，用於金融資料的異常驗證。
透過代理人分工、資料轉換、外部查證與管理層決策，作者嘗試讓 AI 模型模擬人類分析師進行資料鑑別的程序，並強調此流程可大幅減少人工工作量。

9. 研究啟示

本研究的多代理人架構為後續以事件為基礎的金融 AI 系統提供了可借鏡之處，主要包括：

事件驅動流程的重要性
本研究透過異常點作為分析切入點，顯示以「事件觸發」為核心的架構能有效降低資料處理量、提升分析焦點，此概念可延伸至其他以金融事件為基礎的模型設計。

結構化資料轉語義輸入的流程設計
研究中提出的 Data Conversion Agent 展示了如何將表格資料與 metadata 轉換為語言模型可理解的語義描述，提供日後處理市場微結構、分 K 線訊號或事件描述時的流程參考。

代理人分工的概念可用於模型行為解讀
透過 Web Research、Institutional Knowledge、Cross-Check 等多代理人分工方式，本研究指出模型在面對不完整或不一致資訊時可透過「角色區隔」增加穩定性。此概念對需要多角度檢視市場事件或解讀模型推論過程的研究具有參考價值。

管理層代理人的共識形成機制
Management Discussion Agents 展示多模型討論並形成一致結論的可能性，為需要模型穩定度測試、結果一致性分析或決策整合的研究提供方法論啟示。

異常事件的知識蒐集與驗證流程
本研究強調資料品質驗證（data validation）在金融任務中的重要性，尤其在高波動事件中，展示了如何透過外部資料檢索與知識庫提升分析可靠度，對日後結合 LLM 進行事件解讀的研究具實務借鏡價值。
